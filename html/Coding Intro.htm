What the hell is going on? (coding intro)
Lamer introduction:
If you're really lame, you wouldn't even know this. A "mod" is a modification to a game, maybe new weapons, different style of play, whatever.
Most games based off any of id software's 3D engines have the capability to run different mods.

Ok, I'm not a lamer, so what am I doing here?
Well this site provides coding tutorials, ie. helpers to let you get started coding your supposedly wonderful mod creation, for either Quake 2 or Quake 3. Of course to make a real good mod you will proabably need new models/etc but I'm sure you can do without those for a while...
With these tutorials we hope to get you started on whatever you want to do in your mod.

I want to get started in Quake 2 coding!
Simple. Go over to the files page and download both files for Quake 2 coding. LCC is the freeware compiler which will "make" your mods, and the Quake 2 DeathMatch only source code will help you make, well, DeathMatch only mods.
The full set of instructions are included with the Quake 2 DeathMatch only source zip file.

I want to get started in Quake 3 coding!
Quake 3 coding is a bit harder than Quake 2, but still good never-the-less. There are already tutorials for this, one for people who own Microsoft Visual C++ and one tutorial for those who don't own MSVC++.
